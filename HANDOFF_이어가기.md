## 3. 다른 컴퓨터 셋업 — 두 가지 경우

### A. 그냥 보기/발표만 → 아무것도 필요 없음
- 라이브 링크(1번) 열거나, `hooking-deck` 레포 클론해서 `index.html` 더블클릭. 영상·이미지 다 내장돼 있음.

### B. 편집·수정하려면 → 이 폴더(`강의_후킹되는법`)를 통째로 가져가기
이 폴더(약 267MB, 원본 이미지·영상 포함)가 **편집 소스**임. 새 컴퓨터에 복사한 뒤:
```bash
# 필요한 것: python3 + Pillow(이미지) , Chrome(PDF·헤드리스) , ffmpeg(영상 재인코딩)
pip install pillow
# 1) 웹덱 미리보기: web_deck/index.html 을 브라우저로 열기 (상대경로라 그냥 열림)
# 2) 수정 → 자체완결 공유본 + Artifact본 생성:
python3 build_share.py
#    → 강의덱_공유용.html (어디서든 열림) + 강의덱_공유용_artifact.html
# 3) PDF:
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless=new --disable-gpu \
  --no-pdf-header-footer --print-to-pdf=후킹강의덱.pdf "file://$PWD/강의덱_공유용.html"
# 4) 배포: hooking-deck 레포에 index.html(=강의덱_공유용.html)·PDF 복사 후 git push
```

### ⚠️ 반드시 알아야 할 함정 — 경로
- `web_deck/deck.config.js` 와 `web_deck/assets/` = **상대경로**라 이식 OK. **렌더·빌드는 이거만 있으면 됨.**
- `brief.json` = **절대경로**(`/Users/bonbon/...`)로 이미지를 가리킴 → **다른 컴퓨터에선 그 경로가 깨짐.**
- 그래서 **내용을 고칠 땐 `brief.json`이 아니라 `web_deck/deck.config.js`를 직접 고치는 게 안전**(상대경로·자체완결). 또는 brief.json의 `/Users/bonbon/...` 경로를 새 컴퓨터 경로로 일괄 치환 후 재생성.

## 4. 파일 지도 (이 폴더 안)
```
web_deck/
  index.html          ← 렌더러(엔진). 슬라이드 '타입'별 조판·애니·모바일 CSS 전부 여기
  deck.config.js      ← 실제 내용(데이터). 상대경로 → ★수정은 여기가 안전
  assets/             ← 이미지·영상 (상대경로로 참조, 자체완결)
brief.json            ← 내용 원본(절대경로 주의). deck.config.js의 소스
build_share.py        ← web_deck → 자체완결 단일 HTML(이미지·영상 data URI 인라인)
build_assets_0724.py  ← 갤러리 몽타주·인스타 크롭·타임라인 썸네일 생성기(1회성)
restructure_0724.py   ← brief 재구성 이력 스크립트(참고용)
발표스크립트_v3_34장.html  ← ★최신 발표 스크립트(34장 싱크, 연출 큐 포함)
노션정리_후킹강의.md       ← 참가자 배포용 정리(장표 요약 + 참고 사이트 하이퍼링크)
후킹강의덱.pdf         ← 배포 PDF
brief.backup_*.json   ← 편집 단계별 백업(되돌리기용)
```

## 5. 현재 상태 (34장) & 구성
- **오프닝(1~6)**: 표지 → 자담치킨 영상 → 자기소개 → 숫자(영상4편 1,290만) → 오길비 → 이런분께
  - *3·4p는 소개→숫자 순으로 스왑됨*
- **자동화 OS(7~16)**: 반복지옥 → 시스템 선언 → PAZX → 결과물 그리드3 → **조직도(14)** → **하네스 흐름(15)** → 4스텝
  - 14p 조직도 = 오케스트레이션(하네스)=부장 proposal-os, 팀장, 스킬(사원) 구조 (진짜 pipeline.md 기준으로 검증됨)
  - 15p 하네스 흐름 = 컨셉→훅→자료→디자인→검수 + 사람 게이트 4개(나 컨펌 ①②③④)
- **스킬 5종(17~25)**: 훅카피(플립) → 컨셉추얼KV(드래그) → 갤러리 → 크리틱디렉터(다섯 눈 칩) → teaser-factory → 티저영상 → ponytail(빅넘버)
- **시행착오·느낀점(26~29)**: 2000번 → 톤학습 연대기 → 실패지침 → 느낀점
- **마무리(30~34)**: webdeck 따라만들기 → 선물(variant) → 추천계정(gojaehyun.go) → 엔딩
- **엔딩 카피**: "멈추게 하세요 — 스크롤을, 시선을, 마음을."
- **모바일 최적화 완료**: 다단→1단, feature 세로스택, 하네스흐름 가로스크롤, 긴장표 내부스크롤. webdeck 렌더러도 동일 적용.
