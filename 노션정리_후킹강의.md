## 강의 뼈대 (4파트)

### 01 · 후킹이란
- 후킹 = **3초 안에 멈추게 하는 것.** 요즘은 누구나 AI로 잘 만들기 때문에, 차별점은 '어디서 멈추게 할까'라는 사람의 한 끗
- 수상작 예시: 자담치킨 '옷이 날개다' (관용구 비틀기)

### 02 · 내가 만든 자동화 OS
- 예전엔 Claude→GPT→Canva→직접 입력의 **반복 지옥** → "진짜 다 자동화 안 되나?"에서 시작
- 흔한 PPT 자동화가 아니라 **'후킹되는 결과물'이 나오는 시스템** (웹앱 PAZX)
- 구조 = 회사: 나(결정) → 부장(proposal-os, 오케스트레이션) → 팀장(기획·리서치·디자인·검수) → 스킬(사원)
  - *오케스트레이션(하네스) = 일 순서를 정하고 스킬들에게 나눠주는 '부장님'*
- **4스텝**: ① 훅 한 줄 → ② 무드 고정(톤·타이포·원화) → ③④ 검사관 검수 → 반복

### 03 · 내 무기, 스킬 5종
> 스킬 = 내가 일하는 방식을 저장한 파일. AI한테 매번 설명 안 해도 내 기준으로 일하게 하는 **'나 사용 설명서'**. 쓸수록 나를 닮아감.

| 스킬 | 뭐 하는가 | 포인트 |
|---|---|---|
| ① 훅카피 | 3초에 꽂히는 한국어 카피 8~12개 → Top3 | 실제 인스타 카피 계정을 물려 학습, 낚시 금지 |
| ② 컨셉추얼KV | 카피를 포스터급 비주얼로 (시안 6개+) | Product Lock — 제품 형태·로고·색 절대 불변 |
| ③ 크리틱디렉터 | 만들면 바로 검사하는 AI 검사관 | 다섯 개의 눈: 정합성·완료·작품성·사실·비주얼 |
| ④ teaser-factory | 제품 사진 1장 → 3D 모션 티저 영상 | 힉스필드 MCP 생성 + Remotion 자막 합성 |
| ⑤ ponytail (추천 플러그인) | 코드를 최소로 짜게 함 | 코드량 -54% · 토큰 -22% · 시간 -27% (제작자 벤치마크) |

- **톤 학습 연대기**: 물빛(데모) → 하나은행 → 노스엣센스 → 리얼베리어 → 아일로 — 덱마다 반려·교정을 지침으로 저장, 갈수록 첫 시안부터 내 결
- **'이건 아니야'를 2,000번** — 실패(예쁜 무드샷만·파란 콜라·둥둥 뜬 이미지·장표마다 딴 색)가 전부 지침이 됨
